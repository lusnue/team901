# 팀 소개 및 위시리스트 추천 페이지

### 이슈 규칙

<aside>

이슈 예시

- #충돌 해결
- README 레이아웃 개선
- 수정 이력 테이블 추가
- 추천 주제 2 문서 작성
- 추천 주제 1 문서 작성
- 팀원 B 소개 문서 작성
- 팀원 A 소개 문서 작성
- 팀 규칙 문서 작성
- README 메인 페이지 작성
- 저장소 초기 세팅
</aside>

### 커밋 메세지

```bash
[#이슈번호] docs: 작업 설명
[#이슈번호] chore: 작업 설명
[#이슈번호] fix: 작업 설명
[#이슈번호] merge: 작업 설명
```

### 브랜치 전략

```bash
main
 └─ 항상 안정 상태

feature/이름/#<이슈번호>
 └─ 작업용 브랜치
```

예:

```
feature/hayan/#4
```

규칙:

```
1) main 에 직접 push 금지
2) 모든 작업은 feature branch 에서
3) main 반영은 PR 통해서만
4) PR 머지 전 최소 1명 리뷰
```

### PR 전략

목표

```
merge 전에 반드시
"내 작업을 남이 읽고 승인하는 과정"을 경험하게 한다
```

PR 템플릿 예시:

```
## 작업 이슈
- #4

## 변경 내용
- member-a 소개 문서 작성
- README 링크 추가

## 확인 포인트
- 커밋 메시지 규칙 지켰는지
- README 링크가 올바른지
- main 과 충돌 없는지
```
